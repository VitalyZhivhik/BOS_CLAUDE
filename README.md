# Система анализа безопасности сервера

Двухкомпонентная система оценки уязвимостей с автоматической корреляцией
по базам **CVE**, **CWE**, **CAPEC** и **MITRE ATT&CK**.

## Требования

- **Python 3.10+** (скачать: https://www.python.org/downloads/)
- **Windows 10** на обеих машинах
- Сетевая связность между машинами (порт 8443 по умолчанию)
- Права администратора рекомендуются для полного анализа
- **Внешние библиотеки НЕ НУЖНЫ** для работы программ
- **PyInstaller** нужен только для сборки EXE (устанавливается через `setup_env.bat`)

## Почему программа работает без установки библиотек

Вся программа использует **только стандартную библиотеку Python**, которая
идёт в комплекте с Python для Windows:

| Модуль | Назначение |
|---|---|
| `tkinter` | Графический интерфейс (GUI) |
| `socket` | Сканирование портов, сетевое взаимодействие |
| `http.server` | HTTP API на сервере |
| `urllib.request` | Отправка данных с атакующего на сервер |
| `subprocess` | Сбор информации об ОС (реестр, сервисы) |
| `threading` | Многопоточность (сканирование, фоновые задачи) |
| `json` | Работа с базами CVE/CWE/CAPEC/MITRE ATT&CK |
| `concurrent.futures` | Параллельное сканирование портов |

**Единственная внешняя зависимость — PyInstaller** — нужна только для сборки EXE.

## Быстрый старт (GUI)

### Шаг 1. Запуск серверного агента (на защищаемом сервере)

```powershell
cd security_assessment
python gui_server.py
```

В графическом интерфейсе:
1. Нажмите **«1. Анализ системы»** — соберёт информацию об ОС, ПО, сервисах, защите
2. Нажмите **«2. Загрузить базы CVE/CWE/…»** — загрузит базы уязвимостей
3. Нажмите **«3. Запустить сервер»** — запустит HTTP API для приёма данных

### Шаг 2. Запуск атакующего агента (на компьютере атакующей команды)

```powershell
cd security_assessment
python gui_attacker.py
```

В графическом интерфейсе:
1. Введите **IP-адрес сервера** и порт API
2. Нажмите **«Проверить связь с сервером»** — убедитесь, что связь есть
3. Нажмите **«Сканировать порты»** — запустит сканирование
4. Нажмите **«Отправить на сервер»** — отправит результаты для корреляции

Результат появится на вкладке **«Ответ сервера»**, а полный HTML-отчёт
сохранится в папке `reports/` на сервере.

### Шаг 3. Просмотр отчёта

На серверном агенте нажмите **«Открыть последний отчёт»** или загляните в папку `reports/`:
- `report_YYYYMMDD_HHMMSS.html` — визуальный HTML-отчёт
- `report_YYYYMMDD_HHMMSS.json` — машиночитаемый JSON-отчёт

### Альтернатива: запуск без GUI (командная строка)

```powershell
# Сервер
python run_server.py

# Атакующий
python run_attacker.py --target 192.168.1.100 --scan-end 10000
```

## Сборка EXE-файлов

### Вариант 1: Через setup_env.bat (рекомендуется)

```powershell
# 1. Создаёт виртуальное окружение и ставит PyInstaller
setup_env.bat

# 2. Активируем окружение
venv\Scripts\activate.bat

# 3. Собираем оба EXE
python build_exe.py
```

### Вариант 2: Вручную

```powershell
# Создаём виртуальное окружение
python -m venv venv
venv\Scripts\activate.bat

# Устанавливаем PyInstaller
pip install pyinstaller

# Собираем
python build_exe.py --all           # Оба агента (папка с зависимостями)
python build_exe.py --all --onefile # Оба агента (один EXE-файл)
python build_exe.py --server        # Только серверный
python build_exe.py --attacker      # Только атакующий
```

## Что содержит отчёт

### 1. Информация о сервере
- ОС, имя хоста, IP-адреса
- Список установленного ПО
- Работающие сервисы и открытые порты
- Обнаруженные СУБД и веб-серверы
- Состояние средств защиты

### 2. Реализуемые атаки (требуют действий)
Атаки, которые **могут быть проведены** в текущей конфигурации.
Для каждой: CVE, CWE, CAPEC, MITRE ATT&CK ID, серьёзность, описание, рекомендации.

### 3. Нереализуемые атаки (защита обеспечена)
Атаки, которые **не имеют смысла** в текущей конфигурации.

**Пример:** ScanOval обнаружил уязвимость к SQL-инъекциям, но на сервере
нет ни одной СУБД — атака нереализуема.

### 4. Сводные рекомендации по защите
Агрегированный список мер для повышения безопасности.

## Логика корреляции

Система проверяет каждый вектор атаки по следующим критериям:

| Атака | Отмечается нереализуемой, если... |
|---|---|
| SQL-инъекция | Нет СУБД на сервере |
| XSS | Нет веб-сервера |
| EternalBlue | SMB (порт 445) закрыт |
| BlueKeep | RDP отключён (порт 3389 закрыт) |
| PrintNightmare | Print Spooler не запущен |
| Log4Shell | Java не установлена |
| ProxyLogon | Exchange Server не установлен |
| Zerologon | Сервер не является контроллером домена |
| FTP Brute Force | FTP-сервер не запущен (порт 21 закрыт) |
| SSH Brute Force | SSH-сервер не запущен (порт 22 закрыт) |

## Расширение баз данных

Базы данных хранятся в формате JSON в папке `databases/`. Для добавления новых записей
просто добавьте объекты в соответствующий файл, соблюдая структуру.

### Пример добавления CVE:
```json
{
    "id": "CVE-2024-XXXXX",
    "description": "Описание уязвимости",
    "severity": "HIGH",
    "cvss_score": 8.0,
    "affected_software": ["Software Name"],
    "attack_type": "remote_code_execution",
    "related_cwe": ["CWE-XXX"],
    "related_capec": ["CAPEC-XXX"],
    "related_mitre": ["TXXXX"],
    "requires_service": ["service_name"],
    "requires_port": [8080],
    "prerequisites": ["condition"]
}
```

## Настройка

Все настройки в файле `common/config.py`:
- `SERVER_PORT` — порт API сервера (по умолчанию: 8443)
- `SCAN_PORT_START / SCAN_PORT_END` — диапазон сканирования
- `SCAN_TIMEOUT` — таймаут подключения к порту

## Безопасность

Данная программа предназначена **исключительно для тестирования безопасности
собственной инфраструктуры** в рамках Red Team / Blue Team учений.
Использование для сканирования чужих систем без разрешения является нарушением закона.


Как это работает в твоем коде (Корреляция):
Атакующий присылает вектор (CAPEC). Сервер смотрит на свои внутренние данные (ОС, порты, сервисы) и проверяет по CVE, есть ли в этих сервисах дыры. Если дыра есть, сервер связывает её с CWE и выдает вердикт: «Атака РЕАЛИЗУЕМА», подсвечивая это красным.
