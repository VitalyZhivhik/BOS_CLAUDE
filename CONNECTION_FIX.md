# Исправление проблемы подключения клиента к серверу

## Проблема
Клиент (атакующий агент) не мог подключиться к серверу для анализа.

## Причина
В серверном API (`server/api_server.py`) была некорректная логика проверки готовности сервера:

```python
if not state.ready or not state.system_info or not state.vuln_db:
```

Эта проверка возвращала `503 Service Unavailable` даже когда сервер был готов, что приводило к ошибке подключения.

## Решение
Исправлена логика проверки готовности в `server/api_server.py`:

1. **Разделена проверка** на три отдельных условия
2. **Улучшены сообщения об ошибках** - теперь понятно, какой именно шаг не выполнен
3. **Корректный порядок проверок**:
   - Сначала проверяем `system_info` (анализ системы)
   - Затем проверяем `vuln_db` (базы данных)
   - В конце проверяем `ready` (сервер запущен)

## Изменения в коде

### До:
```python
if not state.ready or not state.system_info or not state.vuln_db:
    parts = []
    if not state.system_info:
        parts.append("анализ системы не выполнен")
    if not state.vuln_db:
        parts.append("базы данных не загружены")
    reason = "; ".join(parts) if parts else "сервер не готов"
    logger.warning(f"[HTTP-IN] POST /analyze от {client_ip}: НЕ ГОТОВ ({reason})")
    self._respond(503, {
        "error": f"Сервер не готов: {reason}",
        "ready": False,
        "hint": "Выполните: 1) Анализ системы, 2) Загрузка баз, 3) Запуск сервера"
    })
    return
```

### После:
```python
if not state.system_info:
    logger.warning(f"[HTTP-IN] POST /analyze от {client_ip}: НЕ ГОТОВ (анализ системы не выполнен)")
    self._respond(503, {
        "error": "Сервер не готов: анализ системы не выполнен",
        "ready": False,
        "hint": "Выполните: 1) Анализ системы"
    })
    return

if not state.vuln_db:
    logger.warning(f"[HTTP-IN] POST /analyze от {client_ip}: НЕ ГОТОВ (базы данных не загружены)")
    self._respond(503, {
        "error": "Сервер не готов: базы данных не загружены",
        "ready": False,
        "hint": "Выполните: 2) Загрузка баз данных"
    })
    return

if not state.ready:
    logger.warning(f"[HTTP-IN] POST /analyze от {client_ip}: НЕ ГОТОВ (сервер не запущен)")
    self._respond(503, {
        "error": "Сервер не готов: сервер не запущен",
        "ready": False,
        "hint": "Выполните: 3) Запуск сервера"
    })
    return
```

## Как проверить исправление

1. **Запустите сервер**:
   ```bash
   python run_server.py
   ```
   Выполните все шаги:
   - Анализ системы
   - Загрузка баз CVE/CWE/CAPEC/MITRE
   - Загрузка базы инструментов
   - Запуск сервера

2. **Запустите атакующего агента**:
   ```bash
   python run_attacker.py -t <IP_сервера> -p <порт>
   ```

3. **Проверьте подключение**:
   - Атакующий агент должен успешно подключиться
   - В логе сервера должно появиться: `[HTTP-IN] НОВЫЙ КЛИЕНТ: <IP>`
   - В логе атакующего: `✔  Связь установлена`

## Часто задаваемые вопросы

### Q: Сервер отвечает 503, что делать?
**A:** Проверьте, выполнены ли все три шага:
1. ✅ Анализ системы (кнопка "1. Анализ системы")
2. ✅ Загрузка баз (кнопка "2. Загрузить базы CVE/CWE/CAPEC/MITRE")
3. ✅ Запуск сервера (кнопка "4. Запустить сервер")

### Q: Клиент подключается, но сразу отключается
**A:** Проверьте:
- Правильно ли указан IP адрес сервера
- Открыт ли порт (проверьте брандмауэр)
- Не блокирует ли антивирус соединение

### Q: Как проверить, что сервер готов?
**A:** В GUI сервера:
- Статус должен быть "● Сервер запущен" (зелёный)
- Кнопка "4. Остановить сервер" должна быть активна
- В логе должно быть: "Сервер ГОТОВ к приёму data"

### Q: Где смотреть логи подключения?
**A:** 
- Сервер: вкладка "📝 Лог" в GUI сервера
- Клиент: вкладка "📜 Системный Лог" в GUI атакующего
- Файлы логов: `logs/server.log` и `logs/attacker.log`
