# Руководство по улучшению системы сканирования и корреляции атак

## Что было улучшено

### 1. Атакующий агент (attacker/attacker_agent.py)

#### Расширенные шаблоны атак
- **Добавлено больше векторов атак** для каждого порта:
  - FTP: +1 вектор (перехват учётных данных в открытом тексте)
  - SSH: +1 вектор (устаревший протокол SSH v1)
  - HTTP: +4 вектора (LFI, RFI, CSRF, загрузка файлов)
  - HTTPS: +4 вектора (POODLE, BEAST, ошибки сертификатов, ROBOT)
  - SMB: +3 вектора (null session, EternalChampion, EternalRomance)
  - RDP: +2 вектора (DesktopBridge, слабое шифрование)
  - PostgreSQL: +2 вектора (SQL-инъекция, выполнение команд)
  - VNC: +1 вектор (обход аутентификации)
  - Redis: +2 вектора (манипуляция конфигурацией, SSRF)
  - HTTP Proxy: +2 вектора (SSRF, эксплуатация AJAX)
  - MongoDB: +2 вектора (NoSQL-инъекция, RCE через JavaScript)

#### Генерация атак на основе ОС
- **Автоматическое определение ОС** по открытым портам
- **Добавление специфичных атак** для Windows и Linux:
  - Windows: SAM database extraction, NTLM hash passback, Kerberoasting
  - Linux: Privilege escalation, cron job exploitation, SUID binary exploitation

### 2. Серверный коррелятор (server/attack_correlator.py)

#### Улучшенная система оценки реализуемости
- **Повышены пороги** для более реалистичной оценки:
  - РЕАЛИЗУЕМА: ≥60 баллов (было ≥70)
  - ЧАСТИЧНО РЕАЛИЗУЕМА: ≥30 баллов (было ≥40)
  - ТРЕБУЕТ АНАЛИЗА: ≥10 баллов (было ≥15)
  - НЕ РЕАЛИЗУЕМА: <10 баллов

#### Новая система баллов:
- **Сетевая доступность**: 30 баллов (было 25)
- **Подтверждение Trivy**: 35 баллов (было 30)
- **Уязвимое ПО обнаружено**: 20 баллов (без изменений)
- **Отсутствие патчей**: 10 баллов (было 15)
- **Слабые средства защиты**: 5 баллов (было 10)

#### Детализированные пояснения
- **Контекстные рекомендации** для каждого типа атаки:
  - SQL-инъекции: параметризованные запросы, WAF, валидация
  - XSS: экранирование, CSP, HttpOnly cookies
  - RCE: sandbox, обновление ПО, ограничение прав
  - Brute force: 2FA, ограничение попыток, блокировка IP

#### Улучшенная генерация рекомендаций
- **Специфичные меры защиты** для каждого типа уязвимости
- **Приоритеты устранения** на основе критичности
- **Рекомендации по мониторингу** для всех уязвимостей

## Как использовать

### Запуск атакующего агента
```bash
python run_attacker.py -t <IP_цели> -p <порт_сервера>
```

### Запуск сервера
```bash
python run_server.py
```

### Проверка результатов
1. Атакующий агент отправляет данные на сервер
2. Сервер выполняет корреляцию с учётом:
   - Конфигурации системы
   - Результатов Trivy (если доступен)
   - Баз данных CVE/CWE/CAPEC
3. Генерируется отчёт с:
   - Списком всех векторов атак
   - Оценкой реализуемости для каждого
   - Подробными рекомендациями по защите
